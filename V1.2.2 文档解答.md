# V1.2.2 文档解答

## 文档总体概览

这份文档是**云深处科技（DEEPRobotics）**为其四足机器人**“绝影 Lite2”**提供的软件接口说明文档（V1.2.2）。

这份文档的核心作用是告诉你：**如何通过网络指令控制机器人，以及如何接收机器人反馈的数据。**

为了让你彻底理解并实现控制，我将内容拆解为**通信基础**、**控制指令**、**数据反馈**和**实现步骤**四个部分来详细讲解。

---

### 一、 通信基础 (地基)

在开始控制之前，你必须知道数据往哪里发，以及数据长什么样。

#### 1. 网络配置
机器人使用 **UDP 协议** 进行通信。你需要配置好IP地址。
*   **感知主机/手柄 → 运动主机（发指令）：**
    *   目标 IP: `192.168.1.120`
    *   目标 端口: `43893`
*   **运动主机 → 感知主机（回数据）：**
    *   目标 IP: `192.168.1.103`
    *   目标 端口: `43897`
*   **字节序：** 小端模式（Little-endian）。

#### 2. 数据包格式 (关键)
文档定义了两种数据包格式：**简单指令**（不带参数）和**复杂指令**（带参数）。

| 格式类型     | 适用场景                  | 数据结构 (12字节头 + 数据)          | 说明   |
| :----------- | :------------------------ | :---------------------------------- | :----- |
| **简单指令** | 开关类指令 (如起立、坐下) | `[Code][ParamSize][Type]`           | Type=0 |
| **复杂指令** | 传输大量数据 (如关节角度) | `[Code][DataLength][Type][Data...]` | Type=1 |

*   **Code (指令码):** 相当于“身份证号”，比如 `0x21010202` 代表“起立/趴下”。
*   **ParamSize:** 后面数据的长度（字节）。如果是简单指令，这里填0。
*   **Type:** 区分是简单指令(0)还是复杂指令(1)。

---

### 二、 控制指令 (如何指挥机器人)

这部分是你作为开发者需要发送给机器人的数据。主要分为三类：手柄指令、运动设置、感知设置。

#### 1. 常用控制指令 (手柄下发)
这是最常用的控制方式。你需要向 `192.168.1.120:43893` 发送**简单指令**。

*   **起立/趴下 (切换)**
    *   **指令码:** `0x21010202`
    *   **参数:** 无 (填0)
    *   *注意：发送一次起立，再发送一次趴下。*

*   **步态切换**
    *   **行走步态:** `0x21010300`
    *   **跑步步态:** `0x21010307`
    *   **楼梯步态:** `0x21010401`

*   **动作演示**
    *   **跳舞:** `0x21010204` (舞步1)
    *   **空翻:** `0x21010205`
    *   **射门:** `0x2101020C` (参数1=左脚, 2=右脚)

#### 2. 摇杆控制 (连续运动)
如果你想让机器人动起来，需要用摇杆指令。**注意：摇杆指令是“持续生效”的**，发一次 `10000`，它会一直转，直到你发 `0` 停止。

*   **前后平移:** `0x21010130`
    *   参数范围: `-32768 ~ 32768`
    *   *注意：`±6553` 区间内为停止（死区），防止手柄漂移。*
*   **左右转向:** `0x21010135`
    *   参数范围: `-32768 ~ 32768`

#### 3. 感知与导航指令
*   **开启避障:** `0x21011102`
*   **开启跟随:** `0x21012103`

---

### 三、 数据反馈 (机器人告诉你状态)

机器人会主动向你（感知主机）上报数据，这些数据是**复杂指令**（Type=1），你需要解析数据包中的 `Data` 部分。

| 上报类型     | 指令码   | 数据内容结构       | 频率/说明                         |
| :----------- | :------- | :----------------- | :-------------------------------- |
| **状态信息** | `0x0901` | `RobotStateUpload` | 50Hz。包含电量、IMU角度、速度等。 |
| **关节角度** | `0x0902` | `RobotJointAngle`  | 包含12个关节的角度 (rad)。        |
| **关节速度** | `0x0903` | `RobotJointVel`    | 包含12个关节的速度 (rad/s)。      |
| **指令信息** | `0x0905` | `RobotCmdUpload`   | 上报当前接收的摇杆值和目标速度。  |

---

### 四、 如何实现 (代码逻辑)

根据文档第2章“开发方式”，你可以使用 C++/ROS 或任何支持 UDP 的语言实现。

#### 1. 实现环境准备
*   **操作系统:** Linux (推荐 Ubuntu 18.04/20.04)
*   **依赖:** `boost::asio` 或原生 `socket` 编程，或者 ROS 的 `udp_socket`。
*   **网络:** 确保你的电脑和机器人在同一局域网，或者通过网线直连并配置静态 IP。

#### 2. 代码实现步骤

**第一步：建立 UDP Socket**
```cpp
// C++ 伪代码
int sockfd = socket(AF_INET, SOCK_DGRAM, 0);
struct sockaddr_in dest_addr;
dest_addr.sin_family = AF_INET;
dest_addr.sin_port = htons(43893); // 目标端口
inet_aton("192.168.1.120", &dest_addr.sin_addr); // 目标IP
```

**第二步：构造并发送“起立”指令**
根据 1.1.1 章节，构造 `CommandHead` 结构体。

```cpp
struct CommandHead {
    uint32_t code;
    uint32_t paramters_size; // 注意文档拼写是 paramters，实际代码需对应
    uint32_t type;
};

CommandHead cmd;
cmd.code = 0x21010202;       // 起立/趴下指令码
cmd.paramters_size = 0;      // 简单指令，无参数
cmd.type = 0;                // 简单指令类型

// 发送
sendto(sockfd, &cmd, sizeof(cmd), 0, (struct sockaddr*)&dest_addr, sizeof(dest_addr));
```

**第三步：接收机器人状态 (监听)**
你需要开启一个线程监听 `192.168.1.103:43897`。

```cpp
// 定义状态结构体 (根据 1.4.1)
struct RobotStateUpload {
    int robot_basic_state;
    int robot_gait_state;
    double rpy[3]; // 欧拉角
    double battery_level; // 电量
    // ... 其他字段
};

// 接收循环
while(1) {
    char buffer[1024];
    int len = recvfrom(sockfd, buffer, sizeof(buffer), 0, ...);
    
    // 1. 先解析前12字节的 Header
    CommandHead* head = (CommandHead*)buffer;
    
    if(head->code == 0x0901 && head->type == 1) { // 状态上报
        // 2. 跳过Header，解析后面的数据
        RobotStateUpload* state = (RobotStateUpload*)(buffer + 12);
        printf("Battery: %f\n", state->battery_level); 
        printf("Roll: %f\n", state->rpy[0]);
    }
}
```

### 五、 常见坑点与建议

1.  **心跳包 (Heartbeat):** 文档提到心跳指令码是 `0x21040001`，发送频率需要大于 1Hz。虽然文档说手柄指令不改变控制逻辑，但在某些自动控制模式下，长时间不发心跳可能会导致机器人报错或断连，建议在控制循环中定期发送。
2.  **摇杆死区:** 发送摇杆指令时，数值在 `±6553` (前后) 或 `±9553` (转向) 范围内，机器人认为是0（不动作）。你的代码里需要处理这个死区，不要把微小的摇杆抖动发给机器人。
3.  **字节对齐:** 在 C++ 中定义结构体时，要注意内存对齐（`#pragma pack`），确保 `double` 类型是 8 字节对齐的，否则解析出来的角度会是乱码。
4.  **权限问题:** 运行代码可能需要 root 权限，或者配置好 udev 规则。

**总结：**
要实现控制，你只需要做三件事：
1.  **Socket** 连上机器人的 IP 和端口。
2.  **打包** 一个 12 字节的 Header (Code, Size, Type)。
3.  **Send** 发送这个 Header。

你可以先尝试写一个最简单的程序，发送 `0x21010202` (起立) 指令，看机器人有没有反应，这是验证环境是否搭建成功的最快方法。





## 对第一步代码的详细解释

这段代码其实是在做一件非常具体的事情：**给机器人写一张“信封”**。

如果把控制机器人想象成**寄信**，这段代码就是在告诉电脑：
1.  **我们要寄快递（网络协议）：** 选顺丰（UDP）还是邮政（TCP）？
2.  **收件人地址（IP）：** 机器人在哪里？
3.  **收件人门牌号（端口）：** 机器人身上的哪个口袋用来接这封信？

下面我用“寄信”的逻辑，把你给的每一行代码拆解开来讲：

### 1. 第一步：准备一个装信的盒子
```c
int sockfd = socket(AF_INET, SOCK_DGRAM, 0);
```
*   **大白话解释：** 电脑说：“我要发数据了，请给我准备一个网络通道（或者叫‘插座’）。”
*   **细节：**
    *   `AF_INET`：意思是使用 IP 地址来找人（就像用家庭住址找人）。
    *   `SOCK_DGRAM`：意思是使用 **UDP** 协议。这是一种“发出去就不管”的协议，就像发短信或者寄平信，发完就完事了，不确认对方有没有收到（这对控制机器人来说，速度快比必须送达更重要）。

### 2. 第二步：拿出一张空白信封
```c
struct sockaddr_in dest_addr;
```
*   **大白话解释：** 定义一个叫 `dest_addr` 的信封模板。`struct sockaddr_in` 就是信封的标准格式。

### 3. 第三步：在信封上写上“收件人类型”
```c
dest_addr.sin_family = AF_INET;
```
*   **大白话解释：** 在信封的第一行写上：**“这是发给互联网（IP网络）的信”**。
*   *注：这通常和第一步的类型保持一致。*

### 4. 第四步：写上“收件人门牌号”（端口）
```c
dest_addr.sin_port = htons(43893);
```
*   **大白话解释：** 在信封上写上机器人的**具体端口号**。
*   **为什么看不懂？**
    *   `htons` 是一个特殊的数学公式（字节序转换）。你不需要懂它，只需要知道**43893**就是机器人的“门铃号码”。
    *   **文档对应：** 这个数字在文档的“1.1 协议格式”里写着：运动主机接收指令的端口是 **43893**。

### 5. 第五步：写上“收件人家庭住址”（IP）
```c
inet_aton("192.168.1.120", &dest_addr.sin_addr);
```
*   **大白话解释：** 在信封上写上机器人的**IP地址**。
*   **关键点：**
    *   `192.168.1.120`：这就是机器人的“家庭住址”。
    *   **文档对应：** 同样在文档“1.1 协议格式”里，目标 IP 是 **192.168.1.120**。

---

### 总结：这段代码到底是什么意思？

这段代码整体的意思是：
**“电脑，我要通过 UDP 协议，给住在 `192.168.1.120` 这个地址、门牌号是 `43893` 的机器人，寄一封信。”**

### 接下来你应该怎么做？

这段代码只是**写信封**（配置网络地址）。要真正控制机器人，你还需要做两件事：

1.  **写信的内容（指令）：** 参考文档里的“1.2 手柄下发指令集”。
    *   比如你想让机器人起立，信的内容（指令码）应该是 `0x21010202`。
2.  **把信扔出去（发送）：** 代码里还需要加一行 `sendto(...)`。
    *   这行代码会拿着刚才写好的“信封”（`dest_addr`），把“信的内容”（指令）一起扔给机器人。

**简单操作建议：**
如果你现在想测试，可以先不管这段底层代码。很多开发者会直接使用 Python 或 ROS（一个机器人操作系统）来简化这个过程，直接输入 IP 和端口，然后发送文档里的指令码（如 `0x21010202`）即可。
